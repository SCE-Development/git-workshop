Why Git?

  1. Version Tracking:
    • Git allows you to track changes to your code over time.
    • You can easily compare versions to understand what was added, modified, or deleted.
  2. Collaboration:
    • Git facilitates collaboration among multiple developers on the same project.
    • Multiple people can work on different branches and merge their changes seamlessly.
  3. Branching and Merging:
    • Git makes it easy to create branches for different features or bug fixes.
    • You can merge branches back into the main codebase when they are ready.
  4. Undo and Rollback:
    • Git allows you to undo changes or roll back to a previous state of your code.
    • This can be invaluable for fixing mistakes or dealing with unexpected issues.
  5. Security and Data Integrity:
    • Git ensures data integrity through cryptographic checksums (SHA-1 hashes) for every file and commit.
    • It's difficult for unauthorized users to alter the history or codebase.
  6. History and Documentation:
    • Git's commit history provides a detailed record of all changes, including who made them and why.
    • This serves as valuable documentation for project evolution.
  7. And more!

TL;DR: Git version control offers a robust and flexible way to manage code, collaborate with others, and maintain the integrity and history of your projects, making it an essential tool for developers and teams of all sizes.

Git Cheat Sheet

git init
  • Initializes a new Git repository in the current directory.
git clone [repository URL]
  • Creates a copy (clone) of a remote Git repository to your local machine.
git add [file(s)]
  • Stages changes for commit. You can specify specific files or use "." to add all changes.
git commit -m "[commit message]"
  • Records staged changes in a commit with a descriptive message.
git status
  • Shows the status of your working directory, including untracked, modified, and staged files.
git log
  • Displays a history of commits in the repository, including commit hashes, authors, dates, and messages.
git branch
  • Lists all branches in the repository and indicates the currently active branch with an asterisk (*).
git checkout [branch or commit]
  • Switches to a different branch or a specific commit. Helpful for navigating your project's history.
git pull
  • Fetches changes from the remote repository and merges them into the current branch (usually the default branch).
git push
  • Pushes your local commits to the remote repository, keeping it up to date.
git merge [branch]
  • Merges changes from a specified branch into the current branch.
git remote -v
  • Lists the remote repositories linked to your local repository, showing their URLs.
git fetch
  • Fetches changes from the remote repository but doesn't automatically merge them into the current branch.
git reset [file]
  • Unstages changes from the staging area, or use with "--hard" to reset to a previous commit (cautiondata loss).
git rm [file]
  • Removes a file from the working directory and stages the removal for the next commit.
git diff
  • Shows the differences between the working directory and the last committed version.
git tag [tag name]
  • Creates a tag (a named pointer to a specific commit) for easy reference to important commits.
git stash
  • Temporarily saves changes that aren't ready for commit, allowing you to switch branches or perform other tasks.
git push origin --tags
  • Pushes tags to the remote repository, sharing important points in the project's history.
